Canonical URL & Tagging Reference

This page provides guidelines for canonical URL structure and content tagging standards in HHS content management. Consistent URLs and tagging enable better analytics and SEO.

Canonical URL Structure

Best practices for URL structure:

  • Consistent Format: Use clear, descriptive URL paths
  • Lowercase: All URLs should be lowercase
  • Hyphens: Use hyphens to separate words
  • Canonical Tags: Include canonical tags to prevent duplicate content

Tagging Standards

Required Tags

  • Content Type
  • Primary Topic
  • Target Audience

Optional Tags

  • Related Topics
  • Keywords
  • Department/Team
  • Campaign or Initiative

Tagging Best Practices

  • Use consistent terminology across all content
  • Avoid overly specific or overly generic tags
  • Limit number of tags (3-7 recommended)
  • Review and update tags periodically

Analytics Impact

Proper URL structure and tagging enable:

  • Better SEO performance tracking
  • Content grouping and analysis
  • Duplicate content detection
  • URL-based reporting and filtering
