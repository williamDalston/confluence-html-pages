Fact Tables (PBIX Mock)

This page documents our standard fact table structures and templates. Fact tables contain the measurable, quantitative data at the center of our star schema models.

Fact Table Principles

  • Each row represents a single business event or transaction
  • Contains foreign keys linking to dimension tables
  • Includes numeric measures (metrics) that can be aggregated
  • Should be at the finest grain (detail level) needed for analysis

Standard Fact Tables

Fact_TicketHistory

  • Records ticket status changes and transitions
  • Dimensions: Date, Ticket, Team, Priority, Status, Assignee
  • Measures: Time in Status, SLA Compliance, Ticket Count

Fact_PageViews

  • Records individual page view events
  • Dimensions: Date, Content, User Segment, Traffic Source
  • Measures: Page Views, Unique Visitors, Bounce Rate

Fact_SocialEngagement

  • Records social media engagement events
  • Dimensions: Date, Platform, Content Type, Campaign
  • Measures: Likes, Shares, Comments, Impressions

Template Usage

Use these fact table structures as templates when creating new data models. Ensure consistency in naming conventions, key structures, and measure definitions.
