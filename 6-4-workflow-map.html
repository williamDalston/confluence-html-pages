Workflow Map (Service Desk, UXD, Analytics)

This page documents the workflow and ticket flow between Service Desk, UXD, and Analytics teams, showing how tickets move through different stages and teams.

Workflow Overview

The workflow maps how tickets flow between teams:

  • Service Desk: Initial ticket creation and triage
  • UXD: User experience and design tasks
  • Analytics: Data and reporting requests

Workflow Stages

  1. Ticket Creation: Service Desk receives request
  2. Triage: Assign to appropriate team (UXD or Analytics)
  3. In Progress: Team works on ticket
  4. Review: Work reviewed before completion
  5. Done: Ticket completed and closed

Handoff Points

  • Service Desk → UXD: Design and user experience requests
  • Service Desk → Analytics: Data and reporting requests
  • UXD → Analytics: Analytics requests for design validation
  • Analytics → UXD: Dashboard design collaboration

SLA Tracking

Each stage has defined SLAs:

  • Response time (first response to ticket)
  • Resolution time (ticket creation to completion)
  • Time in each status
