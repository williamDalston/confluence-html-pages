API Token Governance

This page documents API token governance policies and procedures for managing authentication credentials and API access across HHS Analytics tools.

Token Management

  • Creation: Follow standard procedures for creating API tokens
  • Storage: Store tokens securely in Power BI Service or secret management tools
  • Rotation: Rotate tokens regularly per security policy
  • Revocation: Revoke tokens when no longer needed or compromised

Token Types

  • GA4 API Tokens: OAuth 2.0 authentication
  • Jira API Tokens: Basic authentication tokens
  • Sprinklr API Keys: API key authentication
  • Other Tool Tokens: Tool-specific authentication

Security Best Practices

  • Never commit tokens to version control
  • Use environment variables or secure storage
  • Implement least-privilege access principles
  • Monitor token usage and access logs
  • Set up alerts for unusual activity

Documentation

Document:

  • Token locations and storage methods
  • Token owners and responsible parties
  • Rotation schedules and procedures
  • Access logs and audit trails

Compliance

Ensure token management complies with:

  • HHS security policies
  • Federal cybersecurity requirements
  • API provider terms of service

Updates

Note: This page should be updated regularly as token management procedures change or new tools are added. Review and update at least quarterly.
