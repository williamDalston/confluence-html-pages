My Role (Power BI / Data Modeling)

Power BI & Analytics Lead
This section outlines the role and responsibilities of the Power BI & Analytics Lead as the main data analytics / Power BI / modeling expert on the ASPA HHS project.

Key Responsibilities

  • Power BI Dashboard Development: Build executive KPI dashboards for HHS leadership (KPI dashboards, content quality, service desk analytics)
  • Star Schema Data Modeling: Design and implement star schema models for optimal query performance and maintainability
  • Data Source Integration: Integrate Jira, Google Analytics 4, Sprinklr, Drupal, and other sources into Power BI
  • Unified Analytics Workflows: Help unify disparate analytics workflows across content/design/service desk teams
  • Analytics Expertise: Serve as the analytics expert that other teams rely on
  • Reporting Cadence: Eventually take over regular analytics reporting cadence (monthly / quarterly) for HHS leadership
  • Documentation: Maintain comprehensive documentation of all models, datasets, and processes in Confluence

Focus Areas & Expertise

  • Star Schema Modeling: Design optimal data models for Power BI performance
  • DAX Pattern Development: Create reusable DAX formulas and measure patterns
  • M Scripts (Power Query): Build data transformation pipelines and API connections
  • Data Refresh Strategies: Optimize refresh schedules and performance tuning
  • API Integration: Connect to REST APIs (Jira, Sprinklr, GA4) and handle authentication
  • Template Creation: Develop reusable templates for reports, models, and patterns

Key Collaborations

  • Screaming Frog & Metadata: Website crawl data, metadata extraction
  • Social Media Analytics: Sprinklr integration, social follower tracking
  • BigQuery / Looker Engineering: Coordinate Power BI with Looker pipelines
  • Service Desk + UXD Metrics: Ticket analytics, website quality metrics
  • Architecture & DevOps: Infrastructure, authentication, data pipelines
  • Personas & Audience: Segmentation dashboards and taxonomy alignment

Project Goals

  • Build unified analytics system bringing together BigQuery, Looker, Sprinklr, Jira, Screaming Frog, and SiteImprove data
  • Create executive-level KPI dashboards for HHS leadership
  • Establish monthly/quarterly reporting cadence for government stakeholders
  • Reduce manual work and automate workflows (e.g., Sprinklr API integration, strategic crawl scheduling)
  • Document all processes, models, and workflows for team collaboration and HHS review
