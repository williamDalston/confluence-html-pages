Data Model Diagrams

This page contains visual diagrams of our Power BI data models, illustrating the star schema structure and relationships between fact and dimension tables.

Model Structure

Our data models follow a star schema pattern with:

  • Fact Tables: Central tables containing measurable, quantitative data (e.g., tickets, page views, events)
  • Dimension Tables: Descriptive attributes that provide context (e.g., dates, categories, users, content)

Key Models

  • Jira Ticket History Model: Fact table for ticket metrics with dimensions for dates, teams, priorities, and statuses
  • Web Analytics Model: Fact table for page views and events with dimensions for content, dates, and user segments
  • Social Analytics Model: Fact table for social engagement with dimensions for platforms, dates, and content types
