Analytics Team Map

This page provides an overview of the analytics team structure and key collaborators within the ASPA HHS Analytics organization. Understanding team roles, responsibilities, and priorities is essential for cross-functional collaboration.

Core Analytics Team

Will Alston – Power BI & Analytics Lead

  • Primary Focus: Power BI dashboards, star schema data modeling, and unified analytics
  • Responsibilities: Build executive KPI dashboards, integrate data sources (Jira, GA4, Sprinklr), create star schema models, establish monthly/quarterly reporting cadence
  • Key Collaborations: Works closely with all team members to unify disparate analytics workflows

Sogol – Analytics Engineer (BigQuery, Looker, Python)

  • Primary Focus: Google Analytics → BigQuery → Looker pipeline
  • Responsibilities: Maintains GA4 data pipeline, Looker dashboards, Python data engineering
  • Coordination Points: Ensures Power BI and Looker complement each other without duplication
  • Note: Jira → Power BI integration is fine as template exercise, but not strictly necessary

Caroline – Social Media Analytics (Sprinklr Expert)

  • Primary Focus: Sprinklr analytics, social follower tracking, engagement metrics
  • Responsibilities: Daily growth logs, social posts/sentiment insights, Sprinklr API integration
  • Current Workflow: Manual rows.com workflow (targeting automation)
  • Collaboration: Complementary strengths – she's social analytics expert, Will handles data modeling/integration

Stephanie – Metadata / Screaming Frog Specialist (5+ years)

  • Primary Focus: Website crawls using Screaming Frog, metadata extraction
  • Data Extracted: Page metadata, readability scores, canonical URLs, link structure, indexability, GA4 metrics via API
  • Critical Note: Crawls take weeks and consume GA tokens – must be carefully scheduled
  • Upcoming: Creating unified crawl with inputs from entire team

Cross-Functional Partners

Bill – Manager

  • Priorities: Wants dashboards ready once clearance is granted, smooth team integration, initiative without overcommitment
  • Support Areas: Enables access and resource allocation

David – Lead Engineer / Architect

  • Primary Focus: Architecture, DevOps, environment provisioning
  • Expectations: Smart questions about data flows, awareness of authentication/pipelines/system constraints, alignment with DevOps practices
  • Key Role: Will enable technical access and infrastructure

Nathan – Product Manager (Service Desk & UXD)

  • Primary Focus: Service Desk metrics, UXD outcomes, website quality
  • Metrics Needed: Ticket throughput & SLAs, website plain language/readability, UX outcomes
  • Goal: Use analytics to unify chaos across teams

Adrian – Personas & Audience Taxonomy

  • Primary Focus: Audience personas from Qualtrics data, page-topic alignment
  • Collaboration: Provides personas for segmentation dashboards in Power BI

Team Goals & Priorities

  • Unified Analytics System: Bring together data from BigQuery, Looker, Sprinklr, Jira, Screaming Frog, SiteImprove
  • Executive KPI Dashboards: HHS leadership reporting on accessibility, reading levels, performance, social growth, ticket SLAs
  • Reporting Cadence: Establish monthly/quarterly reporting for government stakeholders
  • Automation: Reduce manual work (e.g., Caroline's rows.com workflow, strategic crawl scheduling)
