Cross-Team Data Flow Blueprint

This page documents the data flow blueprint showing how data moves between systems, teams, and analytics platforms across HHS Analytics.

Data Sources

  • Jira: Ticket and workflow data
  • GA4: Website analytics data
  • Sprinklr: Social media analytics
  • Drupal: Content management metadata
  • Screaming Frog: Website crawl data

Data Flow Architecture

Extraction

  1. API calls or connectors extract data from source systems
  2. Data is pulled on scheduled intervals (daily, hourly)
  3. Initial data validation and error handling

Transformation

  1. M scripts in Power Query clean and reshape data
  2. Data is transformed to star schema structure
  3. Business logic and calculations applied

Loading

  1. Data loaded into Power BI star schema models
  2. Dimensions and fact tables updated
  3. Measures and calculated columns refreshed

Team Workflows

  • Engineering: Data infrastructure, API development, pipeline maintenance
  • Analytics: Data modeling, dashboard development, analysis
  • UXD: Dashboard design, visualization standards
  • Content: Metadata standards, content analytics
