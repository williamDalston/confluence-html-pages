Data Sources & Tools Landscape

This page provides an overview of all data sources and analytical tools used across the ASPA HHS Analytics workspace. Understanding this landscape is critical for building unified Power BI dashboards that integrate data from multiple sources.

Primary Data Sources

Website Analytics

  • Google Analytics 4 (GA4): Website traffic, user behavior, engagement metrics, conversion tracking
  • Pipeline: GA4 → BigQuery → Looker (managed by Analytics Engineering team)
  • Power BI Integration: Connect directly to GA4 API or BigQuery for unified dashboards
  • Note: Coordinate with Analytics Engineering team to ensure Power BI and Looker complement each other

Website Crawl Data

  • Screaming Frog: Website crawl data, metadata, SEO metrics, technical audits
  • Data Extracted: Page metadata, readability scores, canonical URLs, link structure, indexability, GA4 metrics via API
  • Owner: Metadata / Screaming Frog Specialist
  • Critical: Crawls take weeks and consume GA tokens – must be strategically scheduled
  • Upcoming: Unified crawl with inputs from entire team

Social Media Analytics

  • Sprinklr: Social media analytics, engagement metrics, follower tracking across Instagram, LinkedIn, Facebook, YouTube
  • Owner: Social Media Analytics team
  • Current Workflow: Manual rows.com workflow (targeting automation)
  • Power BI Goal: Integrate Sprinklr API → Power BI dashboards

Service Desk & Project Management

  • Jira: Ticket tracking, SLA compliance, throughput metrics, team workflows
  • Owner: Service Desk & UXD Product Manager
  • Metrics Needed: Ticket throughput, SLAs, plain language/readability metrics, UX outcomes
  • Note: Jira → Power BI integration is acceptable as template exercise (per Sogol)

Content & Metadata

  • Drupal: Content management system data, metadata, page attributes
  • SiteImprove: Website quality, accessibility metrics, compliance tracking
  • Audience Data: Personas and taxonomy (Qualtrics data)

Analytical Tools & Platforms

Business Intelligence

  • Power BI: Primary BI tool for unified analytics dashboards, reports, and star schema models
  • Looker: Google Analytics dashboards and BigQuery pipelines
  • Coordination: Power BI and Looker should complement each other without duplication

Data Engineering

  • BigQuery: Data warehouse for GA4 data
  • Python: Data pipeline development and automation

Website Analysis

  • Screaming Frog SEO Spider: Website crawling, technical SEO, custom extraction
  • SiteImprove: Website quality, accessibility, and compliance monitoring

Collaboration & Analysis

  • Rows.com: Currently used for manual social follower tracking – targeting automation
  • Microsoft Excel: Ad-hoc analysis and data manipulation

Integration Strategy

The goal is to build a unified analytics system that brings together data from all these sources into Power BI for executive-level KPI dashboards. Key integration points:

  • GA4: Direct API connection or BigQuery integration
  • Jira: REST API with authentication (template exercise)
  • Sprinklr: API integration to replace manual rows.com workflow
  • Screaming Frog: Export crawl results and integrate into Power BI models
  • Drupal: Content metadata via API or exports
